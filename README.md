# 基于出租车GPS数据的城市交通时空特征提取及可视化平台

> **项目定位**：面向城市交通管理的出租车大数据分析与可视化决策支持系统  
> **技术栈**：Python / Django / ECharts / MapV / Baidu Maps API  
> **数据规模**：济南市10个行政区、连续7天、150万+条GPS轨迹记录

---

## 📌 项目简介

本项目针对城市交通拥堵加剧、出租车空驶率高、调度依赖经验等痛点，利用济南市出租车GPS轨迹大数据，构建了一套从**数据清洗 → 特征提取 → 时空聚类 → 关联分析 → 可视化展示**的全链路分析平台。

平台实现了两大核心功能：
- **按时间段查询**城市出租车区域分布（热力图）
- **按车牌号查询**特定车辆的运行轨迹（轨迹回放）

并在此基础上挖掘出6项量化发现，为交通运管部门提供数据驱动的调度决策依据。

---

## 🧩 功能特性

### 1. 数据导入与预处理模块
- 支持 CSV / TXT 格式的GPS原始数据批量导入，自动解析经纬度、速度、方向角、载客状态等字段
- 按日期、车牌号条件筛选导入，支持覆盖导入与增量追加两种模式
- 自动检测并剔除经纬度越界、速度异常、时间戳错乱等异常记录
- 轨迹点去重、漂移点修正，保证轨迹连续性
- 自动识别载客状态切换点，提取OD（上客点/下客点）数据
- 地图匹配：将清洗后的GPS点绑定到济南市路网上
- 支持天气数据、人口数据的导入与关联

### 2. 数据可视化查询模块
- **出租车区域分布查询**：按时间段筛选，百度地图热力图展示济南各区域出租车密度
- **车辆轨迹查询与回放**：输入车牌号和时间范围，动态轨迹线在地图上回放展示
- **OD数据可视化**：上客点/下客点空间分布，按区域聚合OD流量，流向图、弦图呈现
- **多条件组合筛选**：日期、时间段、区域、车牌号等多维度组合查询，支持CSV/Excel导出
- **可视化交互**：地图缩放、拖拽、点击，支持静态视图与动态时间序列演变两种模式

### 3. 数据挖掘分析模块
- **热门上客区域识别**：基于DBSCAN聚类算法（eps=110m, MinPts=30），自动识别城市热点区域
- **乘客需求量时间段分析**：按小时/天/周粒度统计上客量，折线图、柱状图呈现
- **OD订单运营分析**：平均运营时间、平均运营距离、订单频次分布，仪表盘、饼图展示
- **多维度关联分析**：结合人口数据、天气数据，分析不同条件下的出行需求变化规律

### 4. 系统管理模块
- 用户注册、登录、信息修改、密码重置、账号启用/禁用
- 角色权限管理：系统管理员、交通运管人员、数据分析人员、普通用户
- 系统参数配置：数据清洗阈值、聚类参数、可视化效果自定义

### 5. 日志管理模块
- 操作日志：记录用户登录、数据导入、查询、导出等关键操作
- 系统运行日志：数据清洗进度、接口调用状态、地图匹配结果
- 日志在线查询、按条件筛选、导出为CSV/Excel，关键日志永久留存

## 📊 核心发现

### 🔍 发现1：客流时空规律
- **客流峰谷比**：晚高峰21:00（13,036单/小时）、早高峰8:00（12,847单/小时）、凌晨4:00低谷（1,408单）
- **周末 > 工作日**：高出5.9%
- **结论**：济南出租车需求并非纯通勤驱动——夜间休闲出行占比大，周末出行量反超工作日

### 🌦️ 发现2：天气对出行的影响
- 温度与客流相关系数 r = +0.595（暖和天出行更活跃）
- 湿度与客流相关系数 r = -0.554（潮湿天出行减少）
- 降雨与客流相关系数 r = -0.089（数据期内降雨偏少，影响尚不显著）
- **结论**：温度每升1℃，客流约增3.2%

### 📏 发现3：出行距离分布
| 距离区间 | 占比 | 订单数 |
|----------|------|--------|
| 近程 < 3km | 56.2% | 809,762单 |
| 中程 3-10km | 31.3% | 451,407单 |
| 远程 > 10km | 12.5% | 179,706单 |
- **结论**：短途是量的保障（56.2%），远程是利润的来源（12.5%贡献约28%营业额）

### 🚗 发现4：道路拥堵与车速分析
- 平均车速：17.1 km/h
- 拥堵（<14 km/h）：25.9%
- 缓行（14-18 km/h）：35.0%
- 畅通（>18 km/h）：仅39.1%
- 日均载客：143辆，峰值275→低谷2，潮汐现象严重
- **结论**：21:00客流最高但路段不堵——夜间道路充足，车辆数才是运力瓶颈

## 🚀 快速开始

### 环境要求
- Python 3.8+
- Anaconda / PyCharm
- 百度地图API密钥

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/yourusername/taxi-gps-analysis.git
cd taxi-gps-analysis

# 2. 创建虚拟环境
conda create -n taxi-env python=3.8
conda activate taxi-env

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置百度地图API密钥
# 在 web/frontend/static/js/config.js 中填入你的AK

# 5. 数据准备
# 将GPS原始数据放入 data/raw/ 目录

# 6. 运行数据预处理
python preprocessing/gps_cleaner.py
python preprocessing/od_extractor.py
python preprocessing/map_matching.py

# 7. 运行数据分析
python analysis/clustering.py
python analysis/time_pattern.py
python analysis/distance_dist.py
python analysis/weather_correlation.py

# 8. 启动Web服务
cd web/backend
python manage.py runserver

# 9. 打开浏览器访问
http://localhost:8000
```

## 📈 可视化成果展示

### 模块1：上客点热力图
基于DBSCAN聚类将150万条上车点聚合为35-50个热点区域，蓝冷到红热渐变配色，红色区域对应泉城路、万达、火车站等核心商圈。支持缩放漫游和分时段切换。

### 模块2：3D时空散点图
X轴经度、Y轴纬度、Z轴时间/速度，三维空间展示出租车轨迹全貌。散点颜色编码载客状态，鼠标旋转缩放视角。

### 模块3：热力动态图
以1小时为步长自动播放全天24帧客流热力图，底部时间轴支持拖拽跳转和播放暂停，完整呈现潮汐迁移过程。

### 模块4：周客流热力图
7天×24小时矩阵（168个色块），颜色深浅反映客流量高低。周六夜间亮度约为工作日2倍，反常识的周末反高清晰可见。

### 模块5：客流与天气对比
双Y轴设计，同一时间轴叠加客流折线与温度/湿度曲线，温度与客流波动高度同步（r=0.595）。

### 模块6：路程分布堆叠图
近程（<3km，绿色）、中程（3-10km，蓝色）、远程（>10km，橙色）三色堆叠，近程占比稳定56%。

### 模块7：载客车仪表盘
实时显示载客车辆数、拥堵比例、平均车速三大核心KPI，绿黄橙红四级色标预警。

### 模块8：道路速度七日图
周一至周日七条彩色速度曲线叠加，工作日8点和18点出现速度双谷，周末全天速度显著高于工作日。
